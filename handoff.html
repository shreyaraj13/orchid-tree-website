What was built

A fast, premium, mobile-first website built as hand-written HTML, CSS, and vanilla JavaScript on one shared design system. No framework, no build step, so it loads quickly and is simple to maintain. The work covers the full public site, a content engine for SEO, a guest dining journey, staff tooling, a complete brand guideline, and an SEO handover.

Delivered

  • A Plan Your Dining pre-order flow
  • A staff kitchen dashboard
  • A full brand guideline, written and visual
  • An SEO handover guide
  • Source published to GitHub

03

Design system & brand

The complete design language is documented in two places, so anyone can keep new work consistent.

BRAND-GUIDELINES.md

The written reference: colour (including the warm brown family), typography, spacing, every component, motion, photography rules, voice and tone, iconography, SEO standards, data architecture, accessibility, and do/don'ts.

The living style guide, in the browser

A three-page visual guide that renders the real colours, type, and components:

  • Foundations — /brand.html
  • Components — /brand-components.html
  • Voice & Content — /brand-voice.html

House rules, never break these

  • No exclamation marks. No em dashes or en dashes. Use commas and periods.
  • Always write the brand as Orchid Tree, two words.
  • Lead with a feeling, then one specific sensory detail. Keep text sparse, let photos carry.
  • Reach for brown, not grey or black, for any shade, scrim, or shadow.

Search & SEO

The site is built to rank for local, high-intent searches near Bangalore. Full detail is in SEO.md. The headlines:

Already in place

  • Unique title, description, canonical per page
  • Open Graph and Twitter cards
  • Structured data: LodgingBusiness, Restaurant, Service, FAQPage, BlogPosting, Breadcrumb
  • Semantic HTML, clean keyword URLs, alt text
  • Strong internal linking into the money pages
  • Fast, mobile-first, HTTPS

Do first action

  • Optimise the Home page: add a canonical, a meta description, and structured data
  • Add sitemap.xml and robots.txt (ready in SEO.md)
  • Set up Google Search Console and Google Business Profile
  • Decide the live domain and update all URLs, with 301 redirects
  • Set admin and booking pages to noindex

The content engine

The Journal captures informational searches and links into Stays, Restaurant, and Weddings. Aim for two to four quality posts a month. Next keyword ideas: pet-friendly resorts near Bangalore, monsoon getaways, corporate offsite venues near Bangalore, resorts near Whitefield for couples.

05

Making changes & publishing

The site is static and hosted on Vercel. The source lives on GitHub.

To publish changes

Edit the HTML, then run ./deploy.sh from the project folder. It syncs the pages and shared data into the linked Vercel folder and ships to production at orchid-sigma.vercel.app.

Source code

github.com/shreyaraj13/orchid-tree-website (public). Future changes: git add, git commit, git push.

Publishing a new blog post

Duplicate any blog-*.html file, change the title, description, canonical, structured data, and the article body, add a card to blog.html, add the file to deploy.sh, then deploy.

One menu, many surfaces

The dining content lives once in shared/menu.js and feeds the Restaurant page, the Plan Your Dining flow, and the kitchen dashboard. Rooms live once in shared/catalog.js.
