Motion and Interaction

Calm is a design decision

Motion at Orchid Tree is purposeful and restrained. One well-orchestrated reveal beats ten scattered micro-animations. Every rule below was learned from a performance or jank issue on the live site.

Scroll Reveal

Fade and rise

Elements use .reveal + IntersectionObserver. On intersect, add .in. Cascading groups stagger at 0.09s per child. Always include a failsafe that reveals anything still hidden 1.5s after load, so a missed observer never leaves a section blank.

Parallax

Read then write

Full-bleed images drift on scroll via data-parallax + speed. Read all positions first, then write all transforms in a single rAF pass. Skip far-offscreen elements. No layout thrash.

Ken Burns

Slow infinite zoom

Applied to key hero images only. A slow, infinite CSS scale animation adds life without distraction. Disabled entirely under prefers-reduced-motion.

Hover

Image zoom, link grow

Images scale to 1.04 to 1.07 on hover. Nav underlines grow from left via scaleX on ::after. CTA arrows nudge right. Caption overlays reveal. All via CSS transitions, no JS required.

Scroll Progress

2px gold top bar

A thin gold line at the very top of long cinematic pages, driven by a single passive scroll listener. Width mapped to scrollY / documentHeight.

Page Load

One hero reveal

Staggered eyebrow, H1, sub, rule fade-up on DOMContentLoaded. One orchestrated sequence beats scattered micro-animations across the page.

Reduced Motion

Respect the preference

Every animation defers to prefers-reduced-motion: reduce. No Ken Burns, no parallax drift, instant reveals, no scroll-progress bar motion. Test this in OS accessibility settings.

Performance Rules
Critical

No mix-blend-mode overlays

mix-blend-mode on full-screen overlays forces the browser to repaint the entire page every frame. Use semi-opaque fills instead. This rule was added after a measurable scroll-jank incident.

Critical

No stacked backdrop-filter over parallax

Stacking backdrop-filter: blur() on elements that sit over a moving parallax layer is extremely expensive. Use near-opaque fill backgrounds instead of blur for frosted panels over moving images.

Images

content-visibility, decoding, lazy

Use content-visibility: auto on heavy offscreen sections. Add decoding="async" to all images, loading="lazy" below the fold, and fetchpriority="high" on the hero image only.

JS Loop

Single rAF loop

All scroll-driven work (parallax, progress bar, reveal checks) runs inside a single requestAnimationFrame loop with passive event listeners. Never attach multiple independent scroll listeners.

Accessibility

The checklist

WCAG AA minimum. Accessible to every guest, on every device, with any assistive technology. These are requirements, not suggestions.

Colour contrast: WCAG AA

White on olive is approximately 9.6:1. Ink on cream is strong. The olive was chosen precisely for contrast: the earlier forest green (#15583a) was rejected as cool and clinical.

Visible focus states

All interactive elements show a 2px gold outline on :focus-visible. Never use outline: none without a custom focus indicator. Tab through this page to verify.

Keyboard-operable nav and accordions

Nav links and the FAQ accordion are fully keyboard-operable. Escape closes overlays and panels. Focus is managed on open/close. Accordions use aria-expanded and aria-controls.

ARIA usage

aria-current="page" on the active nav link. aria-label on icon buttons and links with no visible text. aria-hidden="true" on decorative icons and the eyebrow lines. aria-expanded on accordions.

Descriptive alt text

Every meaningful image has specific, descriptive alt text. Decorative images use alt="". Placeholders include a comment noting the intended Orchid Tree photo to replace them with.

Reduced motion

All animation respects prefers-reduced-motion: reduce. Ken Burns and parallax are disabled outright. Reveals are instant. Test in System Preferences on macOS or Accessibility settings on iOS.

Semantic landmarks and heading order

Every page uses header, nav, main, section[aria-label], article, and footer. Heading order is strict: one H1, H2 for sections, H3 for cards. Never skip levels.

Touch targets

All interactive elements are at minimum 44px in their tap target. The WhatsApp pill, FAQ triggers, and buttons all meet this. On mobile, stacked buttons never touch.
